# Lenguajes Orientado a Objetos
El enfoque orientado a objetos toma como elemento básico al objeto, el cual es un ente real o abstracto con ciertas características y funcionalidades que lo definen.
## Ventajas
- El desarrollo de aplicaciones es sencillo e intuitivo
- Código elegante
- Objetos completamente independientes

## Programación Orientado a Objetos ***(POO)***
Es un método de implementación en el que los programas se organizan como colecciones cooperativas de objetos. Cada uno representa una instancia de alguna clase, y cuyas clases son miembros de una jerarquía de clases unidas mediante relaciones de herencia.

## Objetivos de la POO 
- **Reutilización:** Capacidad del software de servir para varias aplicaciones
- **Extensibilidad:** Facilidad de adaptarse a cambios de especificación.
- **Corrección:** Capacidad de software para realizar exactamente sus tareas.
- **Robustez:** Capacidad de reaccionar adecuadamente a condiciones excepcionales.

## Elementos fundamentales
La POO tiene cuatro elementos fundamentales:
- **Abstracción:** Permite representar las características esenciales de un objeto.
- **Encapsulamiento:** Es el proceso de almacenar en un mismo compartimiento.
- **Modularidad:** Propiedad que permite subdividir una aplicación en partes más pequeñas.
- **Jerarquía:** Es una clasificación u ordenación de abstracciones. Las dos jerarquías de un sistema complejo son la generalización y la herencia.
Existen tres elementos secundarios:
- **Tipos:** Son una puesta en vigor de la clase de los objetos.
- **Concurrencia:** Es la propiedad que distingue un objeto actitud de uno no activo.
- **Persistencia:** Es la propiedad de un objeto por la que su existencia trasciende el tiempo.

## Clase, Instancia y Objeto 
### Clase
Es una descripción de la organización y las acciones compartidas por uno o más objetos similares.
- Las características comunes a todos los objetos serían sus **atributos**
- Las funcionalidades serían los **métodos**.
### Operación y Método
Una **operación** es una invocación que se hace sobre un objeto, esto es la declaración de un procedimiento, mientras que el **método** es el cuerpo del procedimiento.
### Instancia
Se refiere al hecho de darle un valor específico a cada característica. En este momento podemos decir que se está describiendo la clase. 

### Objetos
De acuerdo con esto, cada objeto es una instancia de su clase y está formando por tantos campos como atributos tiene la misma. Todo objeto tiene estado, comportamiento e identidad.

- Estado: Abarca todas las propiedades del mismo, más los valores actuales de cada una de esas propiedades.
